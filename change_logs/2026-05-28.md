# 2026-05-28

## Non-interactive Prompt Schedule Support

### Goal

Minimize changes to `test_change_prompt.sh` so it runs like the official/non-interactive `test.sh`, while adding only the ability to change prompts according to a pre-defined JSON file. The script no longer needs interactive or web control for prompt changes.

### Changed Files

- `test_change_prompt.sh`
- `hyvideo/generate.py`
- `hyvideo/pipelines/worldplay_video_pipeline.py`
- `assets/prompt_schedule_change_prompt.json`

### Script Changes

`test_change_prompt.sh` now defines:

```bash
PROMPT_SCHEDULE_JSON=./assets/prompt_schedule_change_prompt.json
```

The `torchrun` call remains non-interactive and passes the schedule file to `generate.py`:

```bash
--prompt_schedule_json "$PROMPT_SCHEDULE_JSON"
```

The script also checks that the JSON file exists before starting inference, and stores the schedule path in `run_metadata.json`.

### JSON Format

The prompt schedule JSON uses `segments`. Each segment contains one camera action string and the prompt to use for that segment:

```json
{
  "segments": [
    {
      "pose": "w-46",
      "prompt": "first prompt"
    },
    {
      "pose": "left-41",
      "prompt": "second prompt"
    }
  ]
}
```

`generate.py` concatenates all `pose` values into the same pose string style used by `test.sh`, computes the required `video_length` automatically, and builds a prompt schedule from the segment boundaries.

### Generation Entry Changes

`hyvideo/generate.py` adds:

- `--prompt_schedule_json`
- `load_prompt_schedule_json(...)`

When `--prompt_schedule_json` is provided:

- `args.pose` is overridden by the pose string assembled from JSON segments.
- `args.video_length` is computed from total latent motion duration.
- `args.prompt` is initialized from the first segment prompt.
- The prompt schedule is passed into the pipeline.

### Pipeline Changes

`hyvideo/pipelines/worldplay_video_pipeline.py` now supports an optional AR prompt condition schedule.

For each scheduled prompt:

- prompt embeddings are encoded;
- ByT5 embeddings are prepared when enabled;
- classifier-free guidance negative/positive embeddings are handled consistently with the existing single-prompt path.

During AR rollout, the pipeline selects the active prompt condition by chunk and refreshes the text KV cache when the prompt changes.

### Important Behavior

Prompt changes are applied at AR chunk boundaries. The current AR chunk size is 4 latent frames.

If a segment boundary is not aligned to 4 latent frames, the prompt switch is delayed to the next chunk boundary. This avoids applying the new prompt too early to frames that still belong to the previous segment.

Historical latent selection and camera trajectory generation were not changed.
